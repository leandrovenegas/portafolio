# Plan: Migración de Video a Bunny.net Stream

## Objetivo
Reemplazar el componente `<video>` nativo actual por un reproductor de streaming profesional (Bunny.net Stream) con JSON-LD para SEO y optimización de rendimiento.

---

## Tareas

### 1. Configuración de Variables de Entorno
- Agregar `NEXT_PUBLIC_BUNNY_LIBRARY_ID` en `.env.local`

### 2. Actualizar CSP en next.config.mjs
- Agregar dominios de Bunny.net a las directivas de seguridad:
  - `script-src`: `https://*.bunny.net`
  - `frame-src`: `https://*.bunny.net`
  - `img-src`: `https://*.bunny.net`

### 3. Crear Componente BunnyVideoPlayer
- Aceptar: `videoId`, `title`, `description`, `thumbnail`, `className`
- JSON-LD (VideoObject) inyectado directamente via `next/script`
- Renderizar iframe de Bunny Stream con lazy loading
- Contenedor con `aspect-ratio: 16/9` (evita CLS)
- Estilos: bordes redondeados (`rounded-lg`), sombra (`shadow-lg`), fondo negro
- Controles minimalistas via params de Bunny (theme=minimal o similar)

### 5. Migrar Componente VideoPlayer Existente
- Detectar si la URL es de Bunny.net
- Delegar a BunnyVideoPlayer o mantener comportamiento actual

### 6. Actualizar Usos Existentes
- En `app/proyectos/[slug]/page.js` - actualizar para usar nuevo componente

---

## Estructura de Props Propuesta

```jsx
<BunnyVideoPlayer
  videoId="uuid-de-bunny"
  title="Título del video"
  description="Descripción para SEO"
  thumbnail="https://..."
  className="custom-class"
/>
```

---

## Decisiones Técnicas
- JSON-LD inyectado directamente dentro del componente (no hook separado)
- Controles minimalistas (diseño limpio, sin barra de reproducción invasiva)
